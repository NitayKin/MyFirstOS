GENERAL:
/boot - anything related to booting and the boot sector (ASM)
/kernel - the kernelâ€™s main file, kernel.c, and other kernel related code
/drivers - any hardware specific driver code will go in here.

1) asm(run inside /boot folder):
    nasm main.asm -f bin -o ../boot_sect.bin

2) c(X32, run inside kernel\drivers):
    1.gcc -m32 -masm=intel -fno-pie -ffreestanding -c -mgeneral-regs-only kernel.c -o kernel.o
        to see the output:  objdump -M intel -d kernel.o
        repeat on all C files
    2.ld -m elf_i386 -o ../kernel.bin -Ttext 0xd000 kernel.o --oformat binary
        to see the output:  ndisasm -b 32 kernel.bin > check_file.tmp
        concat the names of every object files that were created on former stage - FIRST ONE SHOULD BE THE KERNEL.O!

3) make the overall image(run inside parent folder):
    cat boot_sect.bin kernel.bin > os-image
    to see the output:  ndisasm -b 32 os-image > check_file.tmp

4) run with qemu:
    qemu-system-x86_64 os-image

5) delete unimportant files ( run on parent folder ):
    find . -name "*.o" -type f -delete
    find . -name "*.bin" -type f -delete
    find . -name "*.tmp" -type f -delete
